# 象棋線上對戰 - 快速開始指南

## ⚡ 30 秒快速開始

### Windows
```bash
start-test.bat
```

### macOS/Linux
```bash
npm run server:build && npm run server
# 然後打開 http://localhost:8080/test.html
```

## 📍 測試頁面地址

```
http://localhost:8080/test.html
```

## 🎮 基本操作

### 1️⃣ 連接到服務器
```
1. 輸入玩家名稱
2. 點擊「連接」按鈕
3. 等待連接狀態變為綠色
```

### 2️⃣ 創建房間
```
1. 點擊「創建房間」按鈕
2. 記下房間 ID
3. 等待其他玩家加入
```

### 3️⃣ 加入房間
```
1. 輸入房間 ID
2. 點擊「加入房間」按鈕
3. 等待房間滿員
```

### 4️⃣ 發送消息
```
1. 點擊「發送測試消息」
2. 查看日誌確認發送
3. 查看「最後接收的消息」確認接收
```

## 🔴 常見問題

### Q: 無法連接？
**A**: 確保服務器已啟動：`npm run server`

### Q: 無法創建房間？
**A**: 確保已連接到服務器（連接狀態為綠色）

### Q: 兩個玩家無法在同一房間？
**A**: 確保使用相同的房間 ID

### Q: 消息無法發送？
**A**: 確保已連接到服務器

## 🛠️ 故障排除

### 連接失敗
```bash
# 1. 檢查服務器是否運行
npm run server

# 2. 檢查地址是否正確
# ws://localhost:8080

# 3. 查看瀏覽器控制台（F12）
```

### 房間操作失敗
```
1. 確保已連接到服務器
2. 確保玩家名稱不為空
3. 查看日誌中的錯誤信息
```

### 消息無法發送
```
1. 確保 WebSocket 連接狀態為「已連接」
2. 查看瀏覽器控制台是否有錯誤
3. 檢查服務器日誌
```

## 💡 提示

1. **使用多個瀏覽器標籤**：在同一瀏覽器中打開多個測試頁面，模擬多個玩家

2. **監控日誌**：日誌會顯示所有操作，便於調試

3. **查看統計信息**：實時監控消息發送和接收情況

4. **檢查最後接收的消息**：確保服務器返回的數據格式正確

5. **使用開發者工具**：按 F12 打開開發者工具，查看 WebSocket 連接
